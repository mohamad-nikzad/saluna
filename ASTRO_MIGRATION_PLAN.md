# Port `apps/web` from Next.js 16 to Astro 6.4.2

> **Astro version targeted: 6.4.2** (latest as of June 2026). Minimum Node: **22.12.0+** (Astro 6 dropped 18 and 20). All package versions and APIs in this plan are verified against the current `docs.astro.build` — see the version table at the end.

## Context

`apps/web` is the public marketing + booking surface (Persian, RTL): landing page (`/`), salon profile (`/salons/[slug]`), booking-status page (`/salons/[slug]/requests/[token]`). It is the only customer-facing SEO target in the repo and is deployed to a Node VPS in Iran (no Cloudflare Workers / no Vercel image optimisation; `images.unoptimized: true` is already set). Both salon routes are `force-dynamic` today, which means every crawler hit, every social-card refetch, and every user click pays a round-trip to the API. We want maximum SEO and minimum TTFB while keeping the same UX.

Astro 6 fits this surface exactly: zero-JS HTML by default, per-route `prerender` control, native `<Image />` from `astro:assets` with stable responsive image support, first-class `@astrojs/sitemap`, React 19 islands via `@astrojs/react@5`, the new **built-in Fonts API** (replaces `next/font` cleanly and works air-gapped), native **CSP**, and `astro:env` for type-safe Zod-validated env vars.

### Confirmed decisions (from user)

| # | Decision | Rationale |
|---|---|---|
| 1 | **Layered rendering**: landing fully static; salon **shell** (name, phone, logo, address, social, bio, theme) SSR with `Cache-Control: s-maxage=…, stale-while-revalidate=…` so nginx serves cached HTML; **services** rendered server-side as SEO HTML on the same response; **availability** stays client-only (existing React island already fetches lazily). | Crawler-visible shell + services without per-request API cost; availability slots remain real-time. This is Astro's equivalent of ISR on a VPS: the Node process renders once, nginx replays the bytes for the next N minutes. |
| 2 | **Keep React islands as-is** via `@astrojs/react@5`. Lift existing TSX components verbatim; mount with `client:load` / `client:visible`. | Fastest port, no rewrite risk, ships React only on routes that need it. React 19 is fully supported. |
| 3 | **Side-by-side**: build new `apps/web-astro` package; both apps run side-by-side until cut-over. | Reversible, easy A/B in nginx, no Big Bang on `main`. |
| 4 | **SEO scope**: sitemap + JSON-LD, dynamic OG images via satori, view transitions. | Sitemap + JSON-LD = highest SEO ROI. Satori OG = social share lift. ClientRouter = UX polish. |
| 5 | **i18n scaffolding deferred**: out of scope; single locale `fa`, RTL. Document where to add later. | Avoid speculative complexity. |

### Why **not** Server Islands for the salon shell

Astro 6 ships Server Islands (`server:defer`) as the canonical "static page + dynamic fragment" primitive. It's tempting for our shell + availability split, but **it would hurt SEO here**: deferred islands render a fallback in the initial HTML and stream the real content after, so crawlers (and many social-card scrapers) miss salon name, services, and JSON-LD. We need that content present in the first response. The right tool for "SSR-once, nginx replays HTML for N minutes" is `Cache-Control: s-maxage=… stale-while-revalidate=…` on a `prerender: false` route — keeping crawler visibility while collapsing per-request API cost.

Server Islands stay in our toolkit for *non-SEO* fragments (e.g., future "your recent bookings" panel keyed off a session cookie). Noted as a follow-up in Phase 6.

### Why this works for SEO

- **Crawlers see real HTML** for landing, salon shell, and services list. Availability is *enhancement*, not content — Google won't punish its absence.
- **`s-maxage` + nginx**: Astro doesn't ship built-in ISR, but the standard VPS pattern (`Cache-Control` + reverse-proxy cache) gives the same effect with simpler ops.
- **JSON-LD `BeautySalon` + `Service[]`** per salon page lets Google build rich results.
- **Per-salon OG images** dramatically increase CTR on Telegram/Twitter/WhatsApp shares.
- **Zero JS on landing**, React only on booking islands.
- **Native CSP** (Astro 6) auto-hashes our inline scripts/styles for a clean `Content-Security-Policy` header without manual nonce wiring.

**4b. Sitemap** (`astro.config.mjs`):
```js
sitemap({
  filter: (page) => !page.includes('/requests/') && !page.includes('/og/'),
  customPages: await loadSalonUrls(),
  serialize(item) {
    if (item.url.includes('/salons/')) {
      return { ...item, changefreq: 'daily', priority: 0.8 }
    }
    return item
  },
})
```
Where `loadSalonUrls()` is a top-level async helper in the config file that hits the public salon list at build time:
```js
async function loadSalonUrls() {
  const base = process.env.PUBLIC_API_URL
  const site = process.env.PUBLIC_APP_URL ?? 'http://localhost:3001'
  if (!base) return []
  try {
    const res = await fetch(`${base}/api/v1/public/salons?limit=1000`)
    if (!res.ok) return []
    const list = await res.json()
    return list.map((s) => `${site}/salons/${s.slug}`)
  } catch {
    return []
  }
}
```
**Open dependency:** the API endpoint to list salons publicly may not exist yet — verify in Phase 0 and either reuse, add to `apps/api`, or skip dynamic injection (fall back to crawl discovery).

**4c. Dynamic OG images** (`src/pages/og/[slug].png.ts`):
```ts
import type { APIRoute } from 'astro'
import satori from 'satori'
import { Resvg } from '@resvg/resvg-js'
import { fetchPublicSalon } from '@/lib/public-api'
import fs from 'node:fs/promises'
import path from 'node:path'

export const prerender = false

const vazirmatn = await fs.readFile(
  path.resolve('./src/assets/fonts/Vazirmatn-Bold.ttf'),
)

export const GET: APIRoute = async ({ params }) => {
  const view = await fetchPublicSalon(params.slug!)
  const svg = await satori(
    /* JSX tree: salon name, theme color band, monogram */,
    {
      width: 1200,
      height: 630,
      fonts: [{ name: 'Vazirmatn', data: vazirmatn, weight: 700, style: 'normal' }],
    },
  )
  const png = new Resvg(svg).render().asPng()
  return new Response(png, {
    headers: {
      'Content-Type': 'image/png',
      'Cache-Control': 'public, s-maxage=86400, stale-while-revalidate=604800, immutable',
    },
  })
}
```
